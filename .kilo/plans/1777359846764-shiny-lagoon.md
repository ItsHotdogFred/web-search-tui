## Proposed Changes

### 1. Add error handling around all external calls
Wrap `exa.search`, `fetch`, `wtf.fetch`, and `client.chat.completions.create` in try/catch blocks. On failure, render a user-friendly error message instead of crashing.

### 2. Extract search-type handlers into separate functions
Break `CreateResultsScreen` into focused functions:
- `handleWikipediaSearch(renderer, query, ...)`
- `handleHelpScreen(renderer, ...)`
- `handleUrlPreview(renderer, url, ...)`
- `handleNativeSearch(renderer, query, ...)`
- `handleWebSearch(renderer, query, ...)`

`CreateResultsScreen` becomes a thin dispatcher (~30 lines).

### 3. Add proper types for renderer and key parameters
Import and use `CliRenderer` (or the appropriate type) from `@opentui/core` instead of `any` for the `renderer` parameter. Type `searchInput` as `InputRenderable`, `searchResults` as `TextRenderable[]`, and `exa` using the Exa client type.

### 4. Extract shared scrollbox rendering helper
Create a `renderResultScrollbox(renderer, results, searchUrls, index, options)` function that handles:
- Creating the `ScrollBoxRenderable`
- Adding result TextRenderables + spacers
- Applying selection highlight
- Adding to renderer.root

Used by both native and web search paths.

### 5. Remove dead code
Delete the unused `positionResults` function.

### 6. Remove unused `resultsStartY` assignments
Either use them or remove the variable entirely.

### 7. Lazy-init OpenAI client
Move the `new OpenAI(...)` call inside the AI summary block (or behind `ENABLE_AI_SUMMARY`) so it only instantiates when needed. Guard with a check for the API key.

### 8. Name layout constants
Define constants at the top of the file:
```ts
const HEADER_Y = 1;
const META_Y = 4;
const DIVIDER_Y = 5;
const CONTENT_START_Y = 6;
```

### 9. Clean up previous scrollbox before re-rendering
At the top of `CreateResultsScreen`, remove any existing `results-scrollbox` and individual search result renderables before creating new ones.

### 10. Normalize string quoting to double quotes
Follow the project convention (most files use double quotes).
